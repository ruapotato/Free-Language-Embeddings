How It Works (V13 — Dual Decoder)

The model compresses English text into 32 concept slots (each a 32-dimensional vector, 1024 dims total). Two non-autoregressive parallel decoders share the same concept bottleneck: (1) EN decoder reconstructs the original English, (2) FR decoder translates to French.

V13 key idea: The French decoder forces language-independent meaning encoding — concepts can’t just store English surface tokens because the FR decoder needs actual meaning to produce French. Word order differs between languages, so the bottleneck must capture structure, not just bags of words.

Trains on 3.2M EN↔FR translation pairs (Europarl + Tatoeba + WikiMatrix, all DFSG-compliant). Every step trains both decoders: EN reconstruction + FR translation from the same pairs.

Training Setup

Architecture 82.8M params — EN encoder (6L) + bottleneck (32×32=1024d) + EN decoder (6L) + FR decoder (6L)
Schedule 600K steps, cosine LR 2e-4 → 1e-5, batch size 48
Data 3.2M EN↔FR pairs: WikiMatrix (2.7M, CC-BY-SA), Tatoeba (275K, CC-BY), Europarl (190K, public domain)
Loss EN recon CE + 1.0 × FR translation CE, both from shared concept bottleneck

Metrics

EN Recon Loss — EN reconstruction cross-entropy. Lower = better.
FR Trans Loss — FR translation cross-entropy. Lower = better translation.
EN Token Acc — Fraction of EN tokens the decoder gets right.
FR Token Acc — Fraction of FR tokens the decoder gets right.
Exact Match — Full EN sentence reconstructed perfectly.
EM EMA — Exponential moving average of exact match (decay=0.9).

Geometry Probes

Analogy — a−b+c≅d cosine score. Want >0.8.
Clustering Gap — Within-group − between-group sim. Want >0.05.
Dir Consistency — Same attribute = same direction? Want >0.3.
Word Order Sim — Swapped-order pair similarity. Want <0.85.
Effective Rank — SVD dims for 90%/95% variance. Higher = richer.

Diagnostic Output

[OK] = perfect reconstruction. [DIFF] (X%) = X% token overlap.
EN diagnostics: prose, code, math, logic. FR diagnostics: Europarl reference translations.
