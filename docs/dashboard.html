How It Works (V15 — Hydra + Geometry)

The model compresses English text into 32 concept slots (each a 16-dimensional vector, 512 dims total). Five non-autoregressive parallel decoders share the same concept bottleneck: (1) EN reconstruction, (2) FR translation, (3) ES translation, (4) Paraphrase, (5) Semantic Parse.

V15 key idea: Combines V14’s multi-head decoder (good for clustering/direction) with V7–V9’s geometry losses (good for word order). V10–V14 never improved word order sim below 0.93 because reconstruction alone doesn’t penalize similar embeddings for different word orders. Geometry losses are recon-gated — they only activate once EM EMA > 0.5, then ramp up over 5K steps.

Trains on 3.2M EN↔FR + EN↔ES pairs (Europarl + Tatoeba + WikiMatrix) plus paraphrase pairs (MRPC/QQP) and semantic parse pairs (custom grammar). Each step: EN recon + one sampled secondary head (25% each) + geometry losses (when gated on).

Training Setup

Architecture ~95M params — EN encoder (6L×384d) + bottleneck (32×16=512d) + 5 decoders (4L×384d each)
Schedule 600K steps, cosine LR 2e-4 → 1e-5, warmup 2K steps, batch size 32
Data 3.2M EN↔FR + EN↔ES pairs, paraphrase pairs (MRPC/QQP), semantic parse pairs
Sampling 25% FR, 25% ES, 25% Paraphrase, 25% Parse (equal weight)
Geometry Gate: EM EMA > 0.5 → ramp 0→1 over 5K steps. WO=2.0, HRepul=3.0, BRepul=1.0
Loss EN recon + secondary head CE + geo_scale × (WO + hard repulsion + batch repulsion)

Metrics

EN Recon Loss — EN reconstruction cross-entropy. Lower = better.
FR/ES Trans Loss — Translation cross-entropy for French and Spanish decoders.
Para Loss — Paraphrase decoder CE. Tests meaning-preserving rewording.
Parse Loss — Semantic parse decoder CE. Tests structural understanding.
EN Token Acc — Fraction of EN tokens the decoder gets right.
Exact Match — Full EN sentence reconstructed perfectly.
EM EMA — Exponential moving average of exact match (decay=0.99). Gates geometry losses.
Geo Gate — Geometry loss scale factor (0–1). Activates when EM EMA > 0.5.

Geometry Probes (every 500 steps)

Analogy — a−b+c≅d cosine score. Want >0.8.
Clustering Gap — Within-group − between-group sim. Want >0.05.
Dir Consistency — Same attribute = same direction? Want >0.3.
Word Order Sim — Swapped-order pair similarity. Want <0.85 (V7–V9 achieved 0.81).
Effective Rank — SVD dims for 90%/95% variance. Higher = richer representations.

Geometry Losses (after gate opens)

Word Order — Push swapped-word pairs below sim 0.5. Weight 2.0.
Hard Repulsion — Push top-8 most similar unrelated pairs below sim 0.1. Weight 3.0.
Batch Repulsion — Push random batch pairs below sim 0.3. Weight 1.0.

Diagnostic Output

[OK] = perfect reconstruction. [DIFF] (X%) = X% token overlap.
EN diagnostics: prose, code, math, logic. FR/ES: Europarl reference translations. Parse: structured output.
