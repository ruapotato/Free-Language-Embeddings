How It Works

The model compresses sentences into 32 concept slots (each a 32-dimensional vector, 1024 dims total). The encoder reads a sentence and produces these concept vectors. The decoder reconstructs the original sentence from the vectors alone. If reconstruction works, the vectors must encode real meaning.

On top of reconstruction, geometry losses shape how the concept space is organized: paraphrases should map to nearby vectors, unrelated sentences should be far apart, and each slot should specialize for one semantic dimension (size, color, tense, sentiment, etc.).

Training Phases

Phase 0 Pure Recon Geo gate = 0. Only reconstruction trains. The decoder learns to read concept vectors. No geometry pressure yet — the model first needs to encode information the decoder can use.
P0→P1 Geo Gate Opens Recon EMA drops enough that the geo gate starts opening. Geometry losses begin flowing, gently at first. Marked with a purple line on charts.
Phase 1 Recon + Geo Ramp Geo gate between 0 and 1. Geometry losses scale with the gate. Decoder gets a weight boost (3x when gate=0, tapering to 1x). Full gradient flows through bottleneck so the encoder learns representations the decoder can actually use.
Phase 2 Slot Focus Geo gate hits 1.0 (recon EMA < 0.1). Cycles through each of the 32 concept slots, 500 steps each. The focus slot gets full gradient (1.0), others get 5%. This guides each slot to specialize for its assigned concept without destroying what other slots learned.
Phase 3 Joint Fine-tune All 32 slots done. All slots train equally. Balanced refinement of the full space. If recon drifts up, the geo gate automatically backs off geometry to protect reconstruction.

The Geo Gate

The geometry gate (geo) is a 0–1 scale that controls how much geometry losses affect training. It’s computed from a smoothed average (EMA) of reconstruction loss:
geo_scale = clamp((0.5 - recon_ema) / (0.5 - 0.1), 0, 1)
When recon is bad (EMA > 0.5), geo = 0 — only reconstruction trains. When recon is good (EMA < 0.1), geo = 1 — full geometry. It’s dynamic: if recon degrades, geometry automatically backs off.

Losses (want LOW)

recon — Reconstruction cross-entropy. Decoder predicts input tokens from concept vectors. Want <0.1 for readable autoregressive output. At 0.5 it’s ~80% per-token accuracy, which cascades into gibberish.
xrecon — Cross-reconstruction. Encode sentence A, decode toward paraphrase B. Tests if concepts capture meaning, not surface form.
nce — InfoNCE contrastive. Pulls paraphrase pairs close, pushes non-matches apart in the batch.
wo — Word-order contrastive. Shuffled sentences should be distinguishable from originals.
decorr — Slot decorrelation. Pushes the 32 slots to encode different, non-redundant information.
iso — Slot isolation. Each slot should only change when its assigned concept (size, color, etc.) changes.
cls — Slot classifier. Auxiliary heads predict concept values from slot vectors.
scon — Slot contrastive. Within each slot: same concept value → close vectors, different value → far apart.
sts — Semantic Text Similarity. Model’s predicted similarity should match human-rated similarity scores.
m_para — Margin: paraphrase. Penalizes paraphrase similarity below 0.92.
m_neg — Margin: negative. Penalizes hard-negative similarity above 0.2.
m_wo — Margin: word-order. Penalizes shuffled-sentence similarity above 0.4.
sp — Per-slot paraphrase on real text pairs. Each slot’s vector should match for paraphrases.
repul — Batch repulsion (mean). Pushes average pairwise similarity of random texts below 0.05.
hrepul — Hard repulsion. Targets the top-k WORST offending pairs per sample, not the diluted mean. The main force pushing the clustering gap wider.

Similarities (want these targets)

p_sim — Paraphrase similarity. Want HIGH (>0.9). Same meaning = close vectors.
n_sim — Hard negative similarity. Want LOW (<0.2). Different meaning = far apart.
wo_sim — Word-order similarity. Want MEDIUM (0.3–0.5). Shared content but different meaning.
sp_sim — Slot paraphrase similarity. Per-slot agreement on real paraphrase pairs.
r_sim — Random batch similarity. Want VERY LOW (<0.05). Random texts should be far apart.
hr_max — Worst-case random pair similarity. The hardest pair to push apart.
cls_acc — Classifier accuracy. Want >0.95. How well slots predict their concept values.
geo — Geometry gate (0–1). How much geometry losses are active right now.

Eval Metrics

clustering_gap — Within-group similarity minus between-group similarity. Want >0.1. Measures if semantically similar sentences actually cluster together.
dir_consistency — Do concept changes (negation, tense) produce consistent vector directions across different sentences? Want >0.5.
rank90/95 — Effective rank of concept space. How many of the 1024 dimensions are actually used (at 90% / 95% variance explained).
slot_iso — Average slot isolation. How well each slot responds only to its assigned concept.
slot_assign — Slots correctly assigned. Want 32/32.
